# Glossary of Operon terms

Short definitions for the words used across these docs. When a term has its own page, follow the link for the full explanation.

## Tasks

- **Inline task**: a Markdown checkbox line that carries Operon metadata, living inside a normal note. See [[DOCS-011 Inline tasks|Inline tasks]].
- **File task**: a Markdown note that is itself a task, with its fields in frontmatter and the body as a working document. See [[DOCS-013 File tasks|File tasks]].
- **operonId**: the durable identity given to every Operon task, so the same task stays recognizable across every surface. See [[DOCS-015 Task identity and operonId|Task identity and operonId]].
- **Parent task / subtask**: a task can contain other tasks, forming a tree. See [[DOCS-016 Parent and sub-tasks|Parent and sub-tasks]].
- **Occurrence**: a single instance generated by a recurring task. See [[DOCS-033 Recurring tasks|Recurring tasks]].

## Fields and customization

- **Canonical field**: an Operon field with one fixed meaning (status, priority, scheduled date, due date, parent task, and so on), reused across every surface.
- **Key mapping**: the link between a canonical field and the property name written in your Markdown, both inline `{{key:: value}}` and file-task frontmatter. See [[DOCS-039 Key mappings|Key mappings]].
- **Custom key**: a canonical key you define yourself when the built-in mappings are not enough. See [[DOCS-040 Custom keys|Custom keys]].
- **Pipeline**: a workflow a task moves through, made of ordered statuses. See [[DOCS-037 Pipelines and statuses|Pipelines and statuses]].
- **Status**: the current stage of a task within its pipeline.
- **Priority**: how important a task is, used for sorting and filtering. See [[DOCS-038 Task priorities|Task priorities]].

## Views and surfaces

- **Filter**: a saved query over task fields that defines which tasks are shown.
- **Filter View**: the surface that displays a filter's results. See [[DOCS-025 Filter View|Filter View]].
- **Dynamic file task filter**: an automatic filter shown inside a file task note, locked to that file's `operonId`, that lists its subtasks. See [[DOCS-026 Dynamic file task filter|Dynamic file task filter]].
- **Task Finder**: fast search to jump to a single task by its text. See [[DOCS-027 Task Finder|Task Finder]].
- **Calendar**: tasks placed on dates and times. See [[DOCS-028 Calendar overview|Calendar overview]].
- **Kanban**: tasks shown as cards in columns by status. See [[DOCS-030 Kanban overview|Kanban overview]].
- **Table**: tasks shown as rows and columns, like a spreadsheet, for comparing, sorting, grouping, and summarizing many fields at once. See [[DOCS-105 Table overview|Table overview]].
- **Table preset**: a saved table setup, its filter, columns, grouping, sorting, and summaries, that you switch between. See [[DOCS-109 Table presets|Table presets]].
- **Table file**: a Table preset stored as its own `.table` file in your vault, which you can open, rename, move, and back up like any note. See [[DOCS-114 Table files|Table files]].
- **Favorite preset**: a preset marked with a star for quick access. Table, Calendar, and Kanban favorites appear as shortcuts in the view's toolbar; saved filters can be favorited too. See [[DOCS-109 Table presets|Table presets]].
- **Column summary**: a value rolled up at the foot of a table column, such as a count, a sum, or the earliest date, shown per group and for the whole table. See [[DOCS-108 Table summaries|Table summaries]].
- **Pinned Task Dock**: a surface that keeps chosen tasks always in view, as a floating dock or a side panel. See [[DOCS-032 Pinned Task Dock|Pinned Task Dock]].
- **Contextual menu**: the right-click and hover menu of task actions, with a configurable, context-aware action list. See [[DOCS-042 Contextual menu actions|Contextual menu actions]].

## Time

- **Time tracking**: recording effort against a task, so duration becomes part of its history. See [[DOCS-034 Time tracking|Time tracking]].
- **TrackTime**: the count-up mode of time tracking, a stopwatch running on a task.
- **FlowTime**: the countdown mode of time tracking, a timed focus session with breaks and an overtime notice. See [[DOCS-035 FlowTime focus sessions|FlowTime focus sessions]].

## Plugin

- **Operon**: the task and project management system these docs describe. The Obsidian plugin id is `operon`.
- **Task Creator**: the guided dialog that builds a task for you. See [[DOCS-020 Task Creator|Task Creator]].
- **Task Editor**: the panel that opens a task to view and change its fields. See [[DOCS-021 Task Editor|Task Editor]].
