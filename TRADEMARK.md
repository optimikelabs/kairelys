# Operon Trademark and Branding Policy

This policy covers the Operon name, logo, icon, visual identity, plugin ID, release identity, and other source-identifying marks associated with the official Operon Obsidian plugin.

The Operon source code is licensed under `GPL-3.0-or-later`. That license does not grant permission to use Operon branding in ways that confuse users about source, sponsorship, approval, or official status.

## Official Use

The official Operon plugin is maintained by Hasan Yılmaz. Official releases, official documentation, and official distribution channels may use the Operon name, plugin ID, icon, and branding.

## Forks and Modified Versions

Modified versions and forks intended for distribution must:

- Use a clearly different project name.
- Use a clearly different Obsidian plugin ID.
- Use a clearly different icon and visual branding.
- State prominently that they are modified, unofficial, and not endorsed by the Operon maintainer.
- Preserve required GPL notices and source availability obligations.

Modified versions and forks must not:

- Present themselves as official Operon releases.
- Use names such as "Operon Pro", "Operon Plus", "Operon Community Edition", or similar names that imply official status or endorsement.
- Use the Operon icon, logo, release artwork, or confusingly similar branding.
- Submit a modified plugin to an ecosystem directory in a way that could reasonably confuse users into believing it is the official Operon plugin.

## Descriptive References

You may refer to Operon descriptively when explaining compatibility, ancestry, or fork origin, for example: "This project is a fork of Operon." Such references must be accurate and must not imply endorsement.

## Permission

For branding uses not covered by this policy, request permission from the maintainer before distribution.
